State Yönetimi Mimarisi
Temel store yapısı ve veri akışı
⚛ Global Store (Zustand / Redux)
// store/index.ts
GlobalStore {
  // Auth
  auth: {
    user: User | null
    token: string | null
    role: Role
    isLoading: boolean
  }

  // Site Ayarları
  settings: {
    meta: SiteMetaState
    theme: ThemeState
    maintenance: MaintenanceState
    i18n: I18nState
    isDirty: boolean
    isSaving: boolean
  }

  // CMS
  cms: {
    posts: Post[]
    currentPost: Post | null
    media: MediaFile[]
    menus: MenuState
    pagination: PaginationState
  }

  // RBAC
  rbac: {
    users: User[]
    roles: RolePermissions
    pendingModeration: Comment[]
  }

  // Analytics
  analytics: {
    liveUsers: number
    pageViews: TimeSeriesData
    topPosts: PostStat[]
    lastUpdated: Date
  }

  // System
  system: {
    logs: LogEntry[]
    performance: PerfMetrics
    cacheStatus: CacheState
    alerts: Alert[]
  }
}
🔄 Veri Akışı
// API → Store → UI akışı

UI Component
  → dispatch(action)
  → Thunk / Saga (async)
    → API call (/api/v1/...)
    → JWT ile auth header
    → Response normalize
  → Reducer state güncelle
  → Selector memoize
  → UI re-render

// Real-time (WebSocket)
WS('/ws/analytics')
  → analytics.liveUsers güncelle
  → analytics.pageViews push

WS('/ws/notifications')
  → system.alerts push
  → rbac.pendingModeration
